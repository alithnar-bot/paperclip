# FS-01 — Artifact Registry and Intake Normalization

## Objective

Add the first persistence slice for factory artifacts and normalize project inputs into a stable planning record.

## Scope

- project-scoped artifact registration
- artifact classification
- intake normalization output
- missing-context candidate generation

## Acceptance

- a project can hold PRD / tech-spec / architecture artifacts
- intake produces a normalized project summary
- unresolved inputs are surfaced explicitly
